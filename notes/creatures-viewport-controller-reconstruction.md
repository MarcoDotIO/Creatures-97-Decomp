# Creatures viewport-controller reconstruction

- `FUN_004022a0` is the small scrollbar resynchronization helper above the viewport object. It resets both scrollbar ranges from the stored page metrics and then snaps the horizontal and vertical scrollbar thumbs back to the midpoint value for each axis.
- `FUN_00403640` is the smoothed recenter helper used when the active tracked target changes. It clears the remembered target when no target is active, rejects targets that have wandered outside the fixed world rectangle, computes the camera origin that keeps the target horizontally centered and vertically anchored at the original five-eighths screen line, and then feeds that origin through the smoothed absolute-seek path.
- `FUN_00403700` is the immediate variant of the same recenter helper. It computes the same anchored origin but routes it through the immediate clamp-and-redraw path instead of the smoothed seeker.
- `FUN_00402680` is the small follow-mode controller above the recovered camera integrator. It first lets the pending smoothed motion finish, then waits for the active target to remain inside the inner follow window for twenty ticks before resynchronizing the scrollbars and promoting the controller into live tracking mode. Once live tracking is enabled, it forwards ticks into the already recovered target-follow helper.
- `FUN_00402700` is the forced-recenter path used when the active target changes abruptly. If viewport following is enabled, it refreshes both scrollbars, promotes the controller directly into live tracking mode, and uses the smoothed recenter helper to bring the viewport back onto the new target before invalidating the whole window.
- The recovered portable implementation lives in `reconstruction/engine/src/creatures_viewport_controller.c` and is covered by the shared slice harness plus the Windows 11 VM slice run.
- `FUN_0040364b` and `FUN_0040370b` still appear to be split-entry decompiler artifacts for the same target-anchor logic, so they remain pending until a live Ghidra session can confirm whether they are true separate entry points or fallthrough aliases.
