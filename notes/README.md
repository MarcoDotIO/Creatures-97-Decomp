# Notes

This directory is the narrative layer of the repository.

It contains:

- subsystem reconstruction notes
- build-verification writeups
- status summaries
- backlog tracking
- collaboration notes for parallel work

## Important Files

- `decomp-status.md`
- `build-verification.md`
- `remaining-functions-to-humanize-and-test.md`
- `shared-workspace.md`

## Naming Convention

Most subsystem notes follow a simple pattern:

- `<area>-reconstruction.md`
- `<area>-reconciliation.md`
- `<area>-preview.md`

The filenames are intentionally explicit so they can stay searchable and can be
mapped back to the reconstructed module or probe target they describe.
